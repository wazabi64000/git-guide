Introduction

Git est un système de gestion de versions distribué qui permet de suivre l’historique des fichiers, collaborer avec d’autres développeurs et gérer efficacement les projets. Que vous travailliez seul ou en équipe, Git est indispensable pour un workflow professionnel.

Travailler en solo

Initialiser un dépôt

# Créer un nouveau dépôt local
git init

# Ajouter tous les fichiers
git add .

# Valider les changements
git commit -m "Premier commit"

Voir l’historique

git log          # Affiche tous les commits
git log --oneline # Résumé compact

Créer des branches

git branch feature1   # Crée une nouvelle branche
git checkout feature1  # Basculer sur la branche
git checkout main      # Revenir sur main

Fusionner une branche

git checkout main
git merge feature1      # Fusionne feature1 dans main

Travailler avec un dépôt distant

git remote add origin https://github.com/utilisateur/projet.git
git push -u origin main  # Pousser la branche principale

Travailler en groupe

Cloner un projet

git clone https://github.com/organisation/projet.git
cd projet

Créer une branche pour une fonctionnalité

git checkout -b feature-nouvelle-fonctionnalite

Ajouter et valider les changements

git add fichier1 fichier2
git commit -m "Description du changement"

Mettre à jour sa branche depuis main

git checkout main
git pull origin main      # Récupérer les dernières modifications
git checkout feature-nouvelle-fonctionnalite
git merge main            # Fusionner main dans votre branche

Résoudre les conflits

Si Git détecte des conflits lors d’un merge, il faut éditer les fichiers concernés, choisir les modifications correctes puis valider :

# Après avoir corrigé les conflits
git add fichier-modifié
git commit -m "Résolution des conflits"

Pousser ses changements et créer une Pull Request

git push origin feature-nouvelle-fonctionnalite
# Puis créer la Pull Request sur GitHub / GitLab / Bitbucket
