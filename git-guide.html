Introduction

Git est un système de gestion de versions distribué puissant, permettant de suivre les modifications de fichiers, collaborer efficacement et revenir sur des versions antérieures. Ce guide couvre les workflows solo et collaboratifs, les erreurs courantes, et les bonnes pratiques pour démarrer un projet proprement.

Préparation avant d’initier un projet

  • Créer un dossier local dédié : Toujours créer un dossier propre pour votre projet avant d’exécuter git init.
  • Fichier .gitignore : Préparer un .gitignore pour exclure fichiers temporaires, logs, secrets, dépendances générées.
  • Initialiser le dépôt : 
    # Dans le dossier du projet
git init   # Crée un dépôt Git local
  • Premier commit : 
    git add .
git commit -m "Initial commit"   # Valider tous les fichiers de départ

Travailler en solo

Créer des branches pour les fonctionnalités

git branch feature1      # Crée une branche
git checkout feature1     # Se positionner dessus
git commit -m "Modifications sur feature1"

Fusionner une branche

git checkout main
git merge feature1   # Fusionner feature1 dans main
# Résolution des conflits si nécessaire

Déployer sur un dépôt distant

git remote add origin https://github.com/user/projet.git
git push -u origin main  # Premier push de la branche principale

Historique et logs

git log            # Historique complet
git log --oneline   # Résumé compact
git status         # Voir modifications en cours

Travailler en groupe

Cloner un dépôt

git clone https://github.com/org/projet.git
cd projet

Mettre à jour sa branche

git checkout main
git pull origin main   # Récupère les dernières modifications
git checkout feature1
git merge main         # Intègre main dans votre branche

Pousser et Pull Request

git push origin feature1
# Créer une Pull Request sur GitHub/GitLab

Erreurs fréquentes et solutions

  • Erreur : "detached HEAD" – Vous êtes sur un commit spécifique et pas sur une branche. git checkout main pour revenir sur la branche.
  • Erreur : "merge conflicts" – Conflits lors d’un merge. Éditer les fichiers conflictuels, git add puis git commit.
  • Erreur : "untracked files" – Fichiers non suivis. Ajouter avec git add fichier.
  • Erreur : push refusé – Votre branche locale est en retard. Faire git pull --rebase puis git push.

Récupérer d’anciennes versions

Revenir à un commit précédent sans perdre l’historique

git checkout   # Voir ancien commit
git checkout main             # Revenir sur la branche principale

Revert : annuler un commit déjà poussé

git revert   # Crée un nouveau commit annulant les changements

Reset : annuler commits locaux

git reset --soft HEAD~1   # Annule le dernier commit mais garde les changements
git reset --hard HEAD~1   # Annule le dernier commit et supprime les modifications

Bonnes pratiques

  • Commits fréquents et messages clairs (git commit -m "fonction X ajoutée").
  • Brancher pour chaque fonctionnalité ou bug.
  • Ne jamais travailler directement sur main en équipe.
  • Tenir à jour le .gitignore.
  • Pull régulièrement pour éviter les conflits.
  • Utiliser Pull Request pour valider les changements en groupe.
  • Tester les merges localement avant push.
